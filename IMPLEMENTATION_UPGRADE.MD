### Phase 1 â€“ Dynamic Service Pages & Navigation Dropdown

This phase introduces dedicated, SEO-friendly detail pages for every service, reachable from a new dropdown menu.

---

#### [NEW] `src/data/services.ts`

Central data module exporting an array of `Service` objects:

```ts
interface Service {
  slug: string;          // URL-safe identifier, e.g. "dental-implants"
  icon: LucideIcon;
  category: 'preventive' | 'cosmetic' | 'restorative' | 'emergency';
  titleKey: string;      // keys into translations for i18n
  descriptionKey: string;
  whyThisTreatment: string;
  forWhom: string;
  guarantees: string;
  whyNotDelay: string;
  faqs: { q: string; a: string }[];
  metaTitle: string;     // SEO
  metaDescription: string;
}
```

- 12 service objects (exam, cleaning, prevention, whitening, veneers, makeover, implants, crowns, rootcanal, emergency, extraction, trauma).
- References translation keys so content stays i18n-aware.

#### [NEW] `src/pages/ServicePage.tsx`

- Uses `useParams()` to read `:slug`.
- Looks up matching `Service` from `services.ts`.
- Renders: hero banner (with **AI-generated service image**), full description, "Why this treatment?", "For whom?", "Guarantees", FAQ accordion.
- CTA to open `BookingModal`.
- "Service not found" fallback if slug is invalid.

#### [NEW] `src/components/ServicesDropdown.tsx`

- Hover (desktop) / tap (mobile) dropdown of all service titles grouped by category.
- Links to `/services/:slug`.
- Keyboard accessible (arrow keys, Escape to close).
- Styled: white bg, `shadow-lg`, `border-gold/20`.

- Add route: `<Route path="/services/:slug" element={<ServicePage />} />`

#### [MODIFY] `src/components/Header.tsx`
